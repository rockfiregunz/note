2-hexo 設定及指令

發表於 2020-02-11 分類於 Hexo

Hexo 全域配置文件

目前只記錄 我用到及理解部分 ，要詳細說明請到 hexo 官網

site

關鍵字 中文 說明
title 網站標題
subtitle 網站副標題
description 網站描述
keywords 關鍵字 使用半型逗號，分隔多個關鍵字
author 作者
language 語系 預設為 en，我設定成台灣 zh-TW
timezone 時區 我設定成台北 Asia/Taipei

Directory

關鍵字 中文 說明
source_dir 原始檔案資料夾，預設:source 這個資料夾用於存放您的內容(建立的文章) ; post = 正式文章，draft = 草稿
public_dir 靜態檔案資料夾 hexo g 產生的檔案會存放在這個資料夾
tag_dir 標籤資料夾
archive_dir 彙整資料夾
category_dir 分類資料夾

Writing

關鍵字 中文 說明
new_post_name 新文章的檔案名稱 new　[layout] “title”
default_layout 預設佈局 new　[layout] “title”
post_asset_folder 文章資料夾，預設:false 建立文章同時建立對應資料夾，我為了區分各文章的檔案資料，所以改成 true
render_drafts 顯示草稿 預設 : false

Extensions

關鍵字 中文 說明
theme 主題名稱 我使用是next

Deployment

關鍵字 關鍵字2 指令
deploy
type git 要發佈的地方
repository git@github.com:帳號/帳號.github.io.git 我使用ssh
branch master git 分支名稱

指令

指令 作用
hexo init 建立 hexo 資料夾
hexo new [layout] “title” 產生文章，layout 沒打就會用預設值
hexo new draft “title” 產生草稿
hexo new publish “草稿標題” 將草稿發布成正式文章，如果要將文章變成草稿，請將post資料夾的文章移到dratf資料夾
hexo generate 產生靜態檔案(public的檔案)
hexo clean 清除靜態檔案(public的檔案)
hexo server 啟動網站 預設：localhost:4000
hexo s -p5000 啟動網站 更改port， localhost:5000
hexo server – draft 啟動網站 ，可觀看草稿
hexo deploy 發佈到網路上

標籤外掛

Markdown 引用檔案的語法 ![檔案標題](/檔案目錄/檔案名稱.檔名)

語法 說明 出處
![](/images/image.jpg) 放在 source 底下的 images 目錄 Markdown
![](image.jpg) 放在文章資料夾的檔案 Markdown
如果使用markdown 本身的語，在首頁會無法呈現，如要呈現，需使用hexo標籤外掛語法 hexo
引用站內文章 hexo

ex:

{% asset_path slug %}
{% asset_img slug [title] %}
{% asset_link slug [title] %}
